# 欠款账单日历管理网页 — 需求规格说明书

## 1. 项目概述

### 1.1 项目目标
构建一个本地运行的 React 网页应用，用于管理欠款账单的还款计划。用户可以通过月历视图直观查看每日待还金额，追踪分期账单的每期还款进度，并获取月度与年度的统计汇总。

### 1.2 产品定位
个人工具类应用，纯前端架构，所有数据通过浏览器 localStorage 进行持久化存储，不依赖任何后端服务或数据库。用户打开浏览器即可使用，无需安装或配置。

### 1.3 目标用户
有信用卡账单、分期贷款、个人借款等还款管理需求的个人用户。用户希望通过日历形式清晰掌握自己每月每日的还款任务。

### 1.4 核心价值
- 一目了然的月历还款视图，避免遗忘还款日
- 分期账单逐期追踪，实时掌握还款进度
- 月度与年度统计，帮助用户规划财务

## 2. 功能需求

### FR-01：单次账单录入
用户可创建单次账单，指定账单名称、金额、还款日期。录入完成后账单出现在日历对应日期上。

- 输入字段：账单名称（文本）、金额（数字）、还款日期（日期选择器）
- 必填校验：名称不能为空，金额必须大于 0，日期不能为空
- 保存后自动刷新日历视图

### FR-02：分期账单录入
用户可创建分期账单，指定账单名称、总金额、分期月数、每月还款日、起始年月。系统根据这些参数自动生成每期记录。

- 输入字段：账单名称（文本）、总金额（数字）、分期月数（数字）、每月还款日（下拉选择 1–28 日）、起始年月（年月选择器）
- 系统自动计算每期应还金额（总金额 ÷ 分期月数），并生成对应期数的还款记录
- 每期记录包含：期次编号、应还金额、还款日期、状态（默认「未还」）

### FR-03：账单编辑
用户可修改已录入账单的所有字段。

- 操作入口：通过日历日期详情或账单管理入口进入编辑
- 编辑后自动更新所有相关日历标注和统计
- 对于分期账单，修改总金额后每期应还金额重新计算；修改分期月数后增加或减少期次

### FR-04：账单删除
用户可删除已录入的账单。

- 删除前弹出确认对话框，防止误操作
- 确认后账单及相关分期记录全部移除
- 日历视图和统计数据同步更新

### FR-05：分期逐期追踪
分期账单的每期可独立标记「已还」或「未还」，并显示还款进度。

- 每期记录显示：期次、还款日期、应还金额、状态
- 用户可点击切换每期的「已还」/「未还」状态
- 进度条显示已还期数 / 总期数及百分比

### FR-06：月历视图
按月份网格显示日历，每个日期上标注该日到期的待还金额。

- 月历格式：7 列网格（周日至周六），按实际月份填充日期
- 日期格子内容包括：日期数字、当日到期待还金额
- 金额颜色规则：已还金额用灰色显示，未还金额用蓝色显示
- 支持切换上一个月和下一个月的导航
- 当月视图默认定位到当前月份

### FR-07：日期详情查看
点击日历中的日期，弹出模态框显示该日所有账单的完整信息。

- 模态框标题显示当前日期
- 列表展示该日到期的所有账单，包含名称、金额、类型（单次/分期）
- 分期账单显示对应的期次编号
- 提供编辑和删除操作入口

### FR-08：月度统计
在日历视图周边显示当月统计信息。

- 当月待还总额：该月所有未还账单金额之和
- 当月已还总额：该月所有已还账单金额之和
- 月均每天待还金额：当月待还总额 ÷ 当月天数
- 统计数值随着用户标记还款状态动态更新

### FR-09：年度统计
以月卡片网格形式展示全年 12 个月的待还与已还汇总。

- 布局：2 行 × 6 列或 3 行 × 4 列的月卡片网格
- 每个卡片显示：月份名称、待还总额、已还总额
- 额外显示年度总待还金额和月均待还金额
- 默认展示当前年份，支持切换年份

### FR-10：数据持久化
账单数据通过浏览器 localStorage 自动保存，页面刷新或关闭后重新打开数据不丢失。

- 每次数据变更（增、删、改、状态切换）后自动写入 localStorage
- 应用加载时自动从 localStorage 读取数据并恢复状态
- 存储键名前缀与应用名称对应，避免与其他应用冲突

### FR-11：空状态引导
首次使用时无账单数据，显示友好的空状态引导提示。

- 日历区域显示提示文字和图标，引导用户添加第一笔账单
- 提供快捷入口跳转到新增账单表单
- 引导文案简洁友好，告知应用的核心用途

## 3. 非功能需求

### NFR-01（性能）
应用启动首屏加载时间在开发模式下小于 1 秒。日历视图渲染 50 笔账单数据时无明显卡顿。

### NFR-02（可靠性）
localStorage 数据格式损坏或读取异常时，应用不崩溃，自动重置为空数据状态，并给出提示。用户操作过程中的数据异常应有容错处理。

### NFR-03（可用性）
界面采用中文，交互直观，常见操作不超过两次点击可完成。无需用户手册即可顺利使用全部功能。表单有适当的校验提示和错误反馈。

### NFR-04（兼容性）
支持 Google Chrome、Microsoft Edge、Mozilla Firefox 的最新稳定版本。在不同浏览器下功能表现一致，样式无明显偏差。

## 4. 数据模型

以下为应用核心数据结构的 TypeScript 接口定义。

### OneTimeBill（单次账单）

```typescript
interface OneTimeBill {
  id: string;            // 唯一标识，使用 UUID 格式，示例："a1b2c3d4-e5f6-..."
  type: "one-time";      // 账单类型标记，固定为 "one-time"
  name: string;          // 账单名称，示例："信用卡还款"
  amount: number;        // 金额，单位元，精度保留两位小数，示例：3500.00
  dueDate: string;       // 还款日期，ISO 格式 "YYYY-MM-DD"，示例："2026-06-15"
  paid: boolean;         // 是否已还，示例：false
  createdAt: string;     // 创建时间，ISO 格式，示例："2026-06-01T10:30:00.000Z"
  updatedAt: string;     // 最后修改时间，ISO 格式，示例："2026-06-01T10:30:00.000Z"
}
```

### Installment（分期账单的某一期）

```typescript
interface Installment {
  periodNumber: number;  // 期次编号，从 1 开始，示例：3
  amount: number;        // 该期应还金额，单位元，示例：1250.00
  dueDate: string;       // 该期还款日期，ISO 格式 "YYYY-MM-DD"，示例："2026-08-15"
  paid: boolean;         // 是否已还，示例：false
}
```

### RecurringBill（分期账单）

```typescript
interface RecurringBill {
  id: string;            // 唯一标识，使用 UUID 格式，示例："b2c3d4e5-f6a7-..."
  type: "recurring";     // 账单类型标记，固定为 "recurring"
  name: string;          // 账单名称，示例："花呗分期"
  totalAmount: number;   // 总金额，单位元，示例：5000.00
  installments: Installment[];  // 分期期次数组
  startDate: string;     // 起始年月，ISO 格式 "YYYY-MM"，示例："2026-06"
  createdAt: string;     // 创建时间，ISO 格式
  updatedAt: string;     // 最后修改时间，ISO 格式
}
```

### Bill（联合类型）

```typescript
type Bill = OneTimeBill | RecurringBill;
```

### BillStore（应用状态）

```typescript
interface BillStore {
  bills: Bill[];         // 所有账单的数组
  currentMonth: string;  // 当前日历视图所在的年月，"YYYY-MM" 格式，示例："2026-06"
  currentYear: number;   // 年度统计视图的年份，示例：2026
}
```

### localStorage 键名约定

| 键名 | 存储内容 | 示例值 |
|------|----------|--------|
| `bill-calendar-data` | 序列化的 BillStore JSON 字符串 | `{"bills":[...],"currentMonth":"2026-06","currentYear":2026}` |

## 5. UI 规范

### 5.1 语言
全界面采用简体中文，包括标签、按钮、提示信息、统计文案、错误消息等所有文本元素。

### 5.2 视觉风格
现代简洁风格，浅色主题。去除冗余装饰，以内容和数据展示为核心。色彩使用克制，以蓝色系作为主色调突出未还金额，灰色系用于已还金额和次要信息。

### 5.3 技术框架
- 样式方案：Tailwind CSS
- 零外部 UI 组件库，所有界面元素（按钮、模态框、日历网格、表单等）均使用 Tailwind 原子类构建
- 不引入日历库或图表库，日历网格和统计展示全部手写实现

### 5.4 页面布局
整体页面采用以下区域划分：

1. **顶部导航栏**：应用名称「欠款账单日历」，月份切换按钮（上个月 / 下个月），年份切换入口
2. **日历主区域**：占据页面大部分空间，展示月历网格，日期格内显示待还金额
3. **统计面板**：位于日历侧边或下方，显示月度待还总额、已还总额、月均每天待还
4. **模态框**：点击日期后弹出，展示该日账单详情，并提供编辑 / 删除操作入口
5. **新增 / 编辑表单**：模态框或页面内表单，用于账单的录入和修改

### 5.5 交互反馈
- 点击可交互元素（按钮、日期格子、账单条目）时有鼠标悬停效果
- 状态切换（已还 / 未还）后立即更新界面显示
- 删除操作前弹出二次确认对话框
- 表单提交前进行字段校验，错误信息直接显示在对应字段下方

## 6. 约束与排除

### 6.1 架构约束
- 纯前端应用，不实现后端服务、数据库或云同步
- 不实现用户登录或多用户功能
- 不实现通知、提醒、邮件或推送功能

### 6.2 功能排除
- 不实现数据导入和导出功能
- 不实现搜索、筛选或排序功能
- 不实现账单分类或标签管理
- 不实现账单逾期计算或逾期提醒

### 6.3 样式排除
- 不实现暗色模式（Dark Mode）
- 不实现移动端自适应布局，桌面优先，分辨率建议 1280 × 720 以上
- 不实现动画过渡效果

### 6.4 依赖排除
- 不引入外部日历库（如 react-calendar、date-fns 等仅日期处理库除外）
- 不引入外部图表库（如 Chart.js、ECharts 等）
- 不引入外部 UI 组件库（如 Ant Design、Material UI 等）

### 6.5 未来可能扩展（当前版本不实现）
本清单仅作为潜在方向记录，不作承诺：
- 数据导出为 CSV 或 JSON 文件
- 还款提醒（基于浏览器 Notification API）
- 账单分类（如信用卡、贷款、个人借款）
- 逾期天数自动计算与标注
